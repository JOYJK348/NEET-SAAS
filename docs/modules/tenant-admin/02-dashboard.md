# Dashboard

## Purpose

The Dashboard provides the Tenant Admin with a complete overview of the institute's daily operations.

It helps the institute owner quickly understand what is happening today, identify urgent tasks, and take immediate action without navigating through multiple modules.

## Objectives

- Provide a quick overview of the institute.
- Highlight today's important activities.
- Show urgent tasks requiring attention.
- Provide shortcuts for frequently used actions.
- Display key institute statistics.

## Dashboard Layout

### 1. Welcome Section

Displays:

- Greeting
- Institute Name
- Current Date
- Academic Year

### 2. Today's Operations

Displays today's scheduled activities.

Information:

- Today's Classes
- Today's Live Classes
- Today's Mock Tests

Actions:

- View Schedule
- View Mock Tests

### 3. Action Required

Displays items that require immediate attention.

Examples:

- Pending Paper Evaluation
- Pending OMR Upload
- Tutor Leave Requests
- Subscription Expiry Alerts

Actions:

- Review Now
- Upload Now
- Manage
- Renew

### 4. Quick Actions

Frequently used shortcuts.

Examples:

- Add Student
- Add Tutor
- Create Mock Test
- Upload Study Material

### 5. Institute Summary

Displays overall institute statistics.

Examples:

- Total Students
- Total Tutors
- Total Batches
- Total Subjects

### 6. Recent Activities

Displays recent activities performed within the institute.

Examples:

- Student Registered
- Tutor Added
- Mock Test Published
- Results Published

### 7. Announcements

Displays important announcements.

Examples:

- Holiday Notice
- Parent Meeting
- System Updates

## Principles

The dashboard should:

- Be simple and easy to understand.
- Focus on today's operations.
- Highlight urgent tasks.
- Provide quick access to common actions.
- Avoid displaying unnecessary information.
- Help the institute owner make quick decisions.
