# Study Materials

## Module Overview

The Study Materials module is responsible for organizing, managing, and distributing all academic learning resources within the coaching institute.

It serves as a centralized repository where tutors publish educational resources and students access the materials required throughout their learning journey.

The objective of this module is to ensure that every student receives the right learning resource at the right time while maintaining a well-organized academic content library.

---

# Objectives

- Centralize all academic learning resources.
- Support classroom and self-learning.
- Organize resources based on the academic structure.
- Enable tutors to publish learning resources efficiently.
- Improve accessibility for students.
- Maintain an organized academic content repository.
- Support continuous academic preparation.

---

# Responsibilities

The Tenant Admin is responsible for managing the institute's learning resource repository.

Responsibilities include

- Organize learning resources.
- Assign resources to courses, batches and subjects.
- Monitor resource availability.
- Manage tutor publishing permissions.
- Maintain academic content quality.
- Archive outdated resources.
- Ensure students receive updated learning materials.

Tutors are responsible for

- Creating learning resources.
- Uploading learning resources.
- Updating existing resources.
- Organizing content for their assigned subjects.

Students are responsible for

- Accessing assigned resources.
- Downloading learning materials.
- Viewing recorded classes.
- Completing published assignments.

# Learning Resource Lifecycle

The complete learning resource journey consists of the following stages.

## 1. Academic Planning

The institute identifies the learning resources required for each academic program.

Purpose

- Plan academic content.
- Support syllabus completion.
- Standardize teaching resources.

---

## 2. Resource Creation

Tutors prepare learning resources.

Examples

- Class Notes
- PDF Documents
- Presentations
- Recorded Classes
- Assignments
- Practice Worksheets
- Previous Year Question Papers

Purpose

- Create quality academic resources.
- Support classroom teaching.

---

## 3. Resource Organization

Learning resources are organized according to the academic hierarchy.

Hierarchy

Course

↓

Batch

↓

Subject

↓

Chapter / Topic (Optional)

↓

Learning Resource

Purpose

- Maintain structured content.
- Simplify navigation.
- Improve discoverability.

---

## 4. Resource Publishing

Learning resources are published for eligible students.

Purpose

- Make resources available.
- Control access.
- Ensure timely distribution.

---

## 5. Student Learning

Students access the assigned learning resources.

Activities

- Read Notes
- Download PDFs
- Watch Recorded Classes
- View Presentations
- Solve Practice Questions
- Complete Assignments

Purpose

- Support classroom learning.
- Improve self-study.
- Enhance examination preparation.

---

## 6. Resource Maintenance

Tutors update learning resources whenever required.

Examples

- Correct mistakes
- Upload revised notes
- Replace outdated files
- Add additional references

Purpose

- Keep academic content up to date.
- Maintain resource quality.

---

## 7. Resource Archive

Completed or outdated resources may be archived.

Purpose

- Preserve academic history.
- Keep active resources organized.
- Reduce unnecessary clutter.

# Access Management

Learning resources should be accessible only to authorized users.

Users

- Assigned Students
- Assigned Tutors
- Tenant Admin

Purpose

- Protect academic content.
- Maintain controlled distribution.

---

# Notifications

The system should notify users whenever new learning resources are published.

Examples

- New Notes Uploaded
- Recorded Class Available
- Assignment Published
- Resource Updated
- Additional Reference Shared

Recipients

- Students
- Tutors

---

# Business Rules

- Every learning resource belongs to a course.
- Every learning resource belongs to a batch.
- Every learning resource belongs to a subject.
- Students can access only assigned resources.
- Tutors can manage only their own published resources.
- Recorded classes should be available only for authorized students.
- Archived resources should remain accessible for administrative purposes.
- Every published resource should support the institute curriculum.
