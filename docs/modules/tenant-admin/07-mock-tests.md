# Mock Tests

## Module Overview

The Mock Tests module is responsible for planning, conducting, evaluating, and analyzing assessments conducted within the coaching institute.

This module enables the institute to measure student performance, monitor academic progress, identify strengths and weaknesses, and continuously improve learning outcomes through structured assessments.

The objective of this module is to provide a complete assessment management system that supports students, tutors, parents, and institute administrators throughout the academic journey.

---

# Objectives

- Plan academic assessments.
- Organize mock tests for different courses and batches.
- Conduct offline and online examinations.
- Evaluate student performance.
- Publish examination results.
- Analyze academic performance.
- Identify weak areas for improvement.
- Support continuous academic growth.
- Monitor institute-wide assessment performance.

---

# Responsibilities

The Tenant Admin is responsible for managing the complete assessment lifecycle within the institute.

Responsibilities include

- Plan institute assessments.
- Create and schedule mock tests.
- Assign tests to batches.
- Coordinate question paper preparation.
- Monitor examination execution.
- Oversee paper evaluation.
- Publish examination results.
- Review institute-wide performance.
- Monitor assessment analytics.
- Plan academic improvements based on assessment outcomes.

---

# Mock Test Lifecycle

The complete assessment journey consists of the following stages.

## 1. Test Planning

The institute plans assessments based on the academic calendar.

Purpose

- Define assessment objectives.
- Select assessment type.
- Prepare examination schedule.

Examples

- Weekly Test
- Chapter Test
- Monthly Test
- Grand Test
- Full Syllabus Test

---

## 2. Question Paper Preparation

Question papers are prepared by the assigned tutors.

Purpose

- Create quality assessments.
- Cover planned syllabus.
- Maintain academic standards.
- Ensure balanced difficulty levels.

---

## 3. Batch Assignment

The mock test is assigned to one or more batches.

Purpose

- Deliver the correct assessment to eligible students.
- Organize examinations batch-wise.

Business Rules

- A mock test may be assigned to multiple batches.
- Students can access only the assessments assigned to their batch.

---

## 4. Test Scheduling

The institute schedules the examination.

Purpose

- Define examination date.
- Define examination time.
- Define duration.
- Coordinate examination activities.

---

## 5. Test Conduct

Students participate in the scheduled examination.

Modes

- Offline Examination
- Online Examination
- Hybrid Examination (Future)

Purpose

- Conduct examinations according to institute policy.
- Ensure smooth assessment execution.
- Maintain examination integrity.

---

## 6. Answer Collection

Student responses are collected after the examination.

Examples

- OMR Sheets
- Written Answer Papers
- Online Submission

Purpose

- Prepare responses for evaluation.
- Maintain assessment records.

---

## 7. Paper Evaluation

Tutors evaluate student responses.

Evaluation Methods

- Manual Paper Evaluation
- OMR Evaluation
- Online Auto Evaluation (Future)
- AI-assisted Evaluation (Future)

Purpose

- Calculate marks.
- Verify responses.
- Provide tutor remarks.
- Measure student understanding.

---

## 8. Result Publication

Evaluation results are published for students and parents.

Purpose

- Share examination outcomes.
- Maintain transparency.
- Support academic review.

---

## 9. Parent Visibility

Parents may access student assessment results through the parent portal.

Purpose

- Improve parent engagement.
- Keep parents informed.
- Encourage collaborative academic improvement.

---

## 10. Performance Analytics

The institute analyzes examination performance.

Examples

- Student Performance
- Batch Performance
- Subject-wise Performance
- Student Ranking
- Top Performers
- Low Performers
- Batch Comparison
- Subject Comparison
- Accuracy Analysis
- Weak Areas
- Strong Areas

Purpose

- Measure academic effectiveness.
- Identify improvement opportunities.
- Monitor institute performance.

---

## 11. Improvement Planning

The institute plans corrective academic actions based on assessment performance.

Examples

- Extra Classes
- Revision Sessions
- Doubt Clearing Sessions
- Parent Meetings
- Individual Mentoring
- Additional Practice Tests

Purpose

- Improve student performance.
- Strengthen weak subjects.
- Enhance learning outcomes.

---

# Business Rules

- Every mock test belongs to a course.
- Every mock test belongs to one or more batches.
- Every mock test should have a defined syllabus.
- Every mock test should have a scheduled date and time.
- Students should participate only in assigned assessments.
- Every completed assessment should be evaluated.
- Evaluation should be completed before publishing results.
- Results should be published only after evaluation.
- Performance analytics should be generated after result publication.
- Improvement activities should be planned based on assessment performance.
- Historical assessment records should always be preserved.
